GitHub Pages Image Fix Guide

Diagnose and resolve image display issues on your site

How to Fix Your GitHub Pages Images

Based on your GitHub Pages site, here are the steps to fix your image display issues:

1

Check Your File Paths

The most common issue is incorrect paths to image files. GitHub Pages is case-sensitive and requires exact paths.

<!-- Correct -->
<img src="images/photo.jpg" alt="Description">

<!-- Incorrect -->
<img src="Images/photo.jpg" alt="Description">
<img src="image/photo.jpg" alt="Description">
2

Verify Case Sensitivity

GitHub Pages is case-sensitive. "Image.jpg" is different from "image.jpg". Check that your file names match exactly.

# Check your repository for exact file names
git status
git ls-files
3

Check File Extensions

Make sure the file extension in your code matches the actual file extension (.jpg vs .jpeg vs .png).

<!-- Match the actual file extension -->
<img src="images/photo.png" alt="Description">
4

Verify Repository Structure

Your images should be in your repository, not just on your local machine. Push all image files to GitHub.

Quick Checklist for Fixing Images

  • Verify all image files are actually in your GitHub repository
  • Check that file paths in your HTML match the case of your folders and files
  • Ensure file extensions match (.jpg, .png, .gif, etc.)
  • Use relative paths that work with your site's structure
  • Clear your browser cache (Ctrl+F5) to see changes
  • Check the browser console (F12) for error messages
Pro Tip: After making changes to your GitHub repository, it may take a few minutes for the changes to appear on your live site.

Need More Help?

If you're still having issues after checking all of the above:

Check Browser Console for Errors

Open Developer Tools (F12) and look for 404 errors in the Console or Network tab.

Verify Files Are Uploaded

Ensure all image files have been committed and pushed to GitHub.

Wait for GitHub Pages Build

Sometimes changes take a few minutes to appear. Check your repository's Actions tab to see if the build is complete.
