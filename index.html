Common Reasons Images Don't Display on GitHub Pages

Based on your GitHub Pages site, here are the most likely reasons some images aren't showing:

1. Incorrect File Paths

The most common issue is incorrect paths to image files. GitHub Pages sites are case-sensitive and require exact paths.

<!-- Correct -->
<img src="images/photo.jpg" alt="Description">

<!-- Incorrect -->
<img src="Images/photo.jpg" alt="Description">
<img src="image/photo.jpg" alt="Description">

2. Case Sensitivity

GitHub Pages is case-sensitive. "Image.jpg" is different from "image.jpg".

3. File Extensions

Make sure the file extension in your code matches the actual file extension (.jpg vs .jpeg vs .png).

4. Repository Structure

Your images should be in your repository, not just on your local machine.

How to Fix Your Images

1

Check Your Repository Structure

Ensure your images are actually in your GitHub repository, not just on your local machine.

Check Your Repository
2

Use Correct Relative Paths

If your HTML file is at the root and your images are in an "images" folder:

<img src="images/your-image.jpg" alt="Description">

If your HTML file is in a folder and you need to go up one level:

<img src="../images/your-image.jpg" alt="Description">
3

Verify Case Sensitivity

Check that the case of your folder and file names matches exactly what's in your code.

4

Clear Your Browser Cache

Sometimes old versions of pages are cached. Press Ctrl+F5 to hard refresh.

Next Steps

If you're still having issues after checking these points:

  1. Open your browser's Developer Tools (F12) and check the Console tab for error messages
  2. Look for 404 errors in the Network tab indicating missing files
  3. Verify your image files are committed and pushed to GitHub
  4. Check that your image files aren't too large (optimize images if needed)
